================================================================================
FUTURE SCALE: HOW TO CONVERT TO MICROSERVICES + EVENT-DRIVEN
================================================================================
Currently, the backend operates as "Microservices (Modular)". Modules act as logical 
services that communicate directly via internal APIs.

As the platform scales to 30-40 Microservices, the architecture will evolve into 
an "Event-Driven Microservices" model:

1. Breakout Modules: Each folder in `business/`, `core/`, and `shared/` will become 
   its own physically independent Node.js/Go service container.
2. Event-Driven Messaging: Instead of synchronous API calls, services will communicate 
   asynchronously via an Event Bus (e.g., Apache Kafka / RabbitMQ). 
   (e.g., The `rider` service emits a "Ride Completed" event, and the `wallet` service 
   listens to that event to move the funds).
3. API Gateway: The `core/api-gateway` becomes the sole entry point for the Frontend 
   and Admin apps, securely routing external HTTP requests to the 40+ internal services.
4. Database Scaling: Currently using a Single Database. In the future, this transitions 
   to "Multiple DB per service" (e.g., the `wallet` service uses a high-consistency Postgres DB, 
   while the `chat` service uses a NoSQL MongoDB).
